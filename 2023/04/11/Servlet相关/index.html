Servlet 创建方式

  • 实现Servlet接口，并重写其全部方法

    • void init(ServletConfig config)

    • void service(ServletRequest req, ServletResponse res)

    • void destroy()

    • ServletConfig getServletConfig()

    • String getServletInfo()

  • 继承GenericServlet抽象类，并重写service()方法

  • 继承HttpServlet抽象类，并重写doGet()或doPost()方法

    HttpServlet针对HTTP/1.1协议中定义的7种请求方式定义7中对应的方法，doGet()、doPost()、doHead()、doPut()、doDelete()、doTrace()和doOptions()，HttpServlet重写了service()方法，会先获取客户端的请求方式，然后根据请求方式调用对应的doxxx()方法。

GenericServlet是实现了Servlet接口的抽象类，HttpServlet是GenericServlet的子类，具有GenericServlet的全部特性。

@WebServlet 注释

注释不需要依赖于配置文件，可以直接在类中使用，其配置仅对当前类有效。@WebServlet用于将一个类声明为Servlet，该注释会在部署时被容器处理，容器将根据其具体的属性配置将对应的类部署为Servlet。

load-on-startup

用来标记Servlet容器启动时是否初始化当前Servlet以及当前Servlet的初始化顺序。

load-on-startup的取值规则：

  • 必须是整数

  • 小于0或未指定时，表示容器在该Servlet首次被请求时才会加载

  • 大等于0时，表示容器在启动时就加载并初始化该Servlet，取值越小，优先级越高

  • 取值相同时，容器自行选择顺序进行加载

Servlet 生命周期

  • 初始化阶段

    • 加载和实例化Servlet： 服务器第一次被访问时加载一个Servlet容器，且只会被加载一次。

    • 调用init()方法初始化： Servlet实例化后，调用一次init()方法，只会被调用一次。

  • 运行时阶段

    响应请求，Web服务器调用service()方法来处理请求，并传入一个请求对象ServletRequest和一个响应对象ServletResponse作为参数。

  • 销毁阶段

    在Servlet生命周期结束时调用destroy()，只会被调用一次。

    在调用destroy()方法后，Servlet对象将被标记，由JVM进行垃圾回收处理。

Servlet 处理请求

  • 客户端发送请求到服务器

  • 容器（Tomcat）根据请求以及web.xml判断对应的Servlet是否存在，存在则继续执行，不存在则返回404

  • 判断对应的Servlet是否已被实例化，若未被实例化，容器将加载相应的Servlet到JVM并实例化。

  • 开启新线程调用实例对象的service()方法，执行相关处理。

  • 返回响应。

Servlet 单例多线程

Servlet是单例多线程的，只会创建一个Servlet对象，每次请求时会起一个线程并在该线程的栈内存中执行service()方法。为了保证线程安全，一般情况下不考虑在Servlet类中定义可修改的成员变量，因为每个线程都将可以修改这个成员变量。

ServletConfig

Servlet容器初始化Servlet时，会为Servlet创建一个ServletConfig对象，并将该对象作为参数传递给Servlet，通过ServletConfig对象可以获得当前Servlet的初始化参数信息。

一个Servlet只能对应一个ServletConfig对象。

获得Servlet对象的方式：

  • 直接从带参的init()方法中获取

  • 调用GenericServlet提供的getServletConfig()方法获得

ServletConfig接口：

  • String getInitParameter(String name)

    根据初始化参数名name返回对应的初始化参数值

  • Enumeration<String> getInitParameterNames()

    返回Servlet的所有初始化参数名的枚举集合

  • ServletContext getServletContext()

    返回一个代表当前Web应用的ServletContext对象

  • String getServletName()

    返回Servlet的名字

ServletContext

Servlet容器启动时，会为每个Web应用（webapps下的每个目录都是一个 Web 应用）创建一个唯一的ServletContext对象，该对象一般被称为“Servlet上下文”。

ServletContext对象的生命周期从Servlet容器启动时开始，到容器关闭或应用被卸载时结束。

Web 应用中的所有Servlet共享同一个ServletContext对象，不同Servlet之间可以通过ServletContext对象实现数据通讯，因此ServletContext对象也被称为Context域对象。

获得ServletContext对象的方式：

  • 通过GenericServlet提供的getServletContext()方法

  • 通过ServletConfig提供的getServletContext()方法

  • 通过HttpSession提供的getServletContext()方法

  • 通过HttpServletRequest提供的getServletContext()方法

ServletContext应用：

  • 获取上下文初始化参数

    通过web.xml中的<context-param>元素可以为web应用设置一些全局的初始化参数，即上下文初始化参数。

    应用中的所有Servlet都共享同一个上下文初始化参数。在 Web 应用的整个生命周期中，上下文初始化参数会一直存在，并且可以随时被任意一个 Servlet 访问。

    ServletContext接口中用于获取上下文初始化参数的方法：

    • String getInitParameter(String name)

    • Enumeration<String> getInitParameterNames()

  • 实现Servlet之间的数据通讯

    ServletContext接口中用于实现数据通讯的方法：

    • void setAttribute(String name, Object object)

      把一个 Java 对象与一个属性名绑定，并将它作为一个属性存放到 ServletContext 中。

    • void removeAttribute(String name)

      从 ServletContext 中移除属性名为 name 的属性。

    • Object getAttribute(String name)

      根据指定的属性名 name，返回 ServletContext 中对应的属性值。

  • 读取Web应用下的资源文件

    可以读取配置文件和日志文件等…

    ServletContext接口中用于读取web资源的方法：

    • Set getResourcePaths(String path)

      返回一个 Set 集合，该集合中包含资源目录中的子目录和文件的名称。

    • String getRealPath(String path)

      返回资源文件的真实路径（文件的绝对路径）。

    • URL getResource(String path)

      返回映射到资源文件的 URL 对象。

    • InputStream getResourceAsStream(String path)

      返回映射到资源文件的 InputStream 输入流对象。
